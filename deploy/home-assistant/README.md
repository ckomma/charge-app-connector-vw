# Home Assistant App

This directory contains Home Assistant App/Add-on deployment notes for the
Volkswagen App Connector. It is an optional deployment method alongside the
systemd and Docker Compose examples.

The installable custom-repository add-on lives in
[`../../addons/vw-app-connector`](../../addons/vw-app-connector). The legacy
local staging flow is still available through `tools/package_ha_app.ps1` for
development and manual HA OS tests.

The app stores usage counters, caches and diagnostics below `/data`, which is
persistent app storage managed by Home Assistant Supervisor.

USB ADB access is enabled through the app configuration. ADB over Wi-Fi remains
the cleaner option when USB device passthrough is not available or unreliable on
the HAOS host.

## Install From The Add-on Store

Add this GitHub repository URL [`charge-app-connector-vw`](../..) as a custom add-on repository in Home Assistant:

```text
Settings -> Add-ons -> Add-on Store -> Repositories
```

After the store reloads, install `Volkswagen App Connector`, configure it, and
start the add-on.

## Prerequisites

- Home Assistant OS or another Supervisor-based installation.
- The Advanced SSH & Web Terminal app or another way to copy files to the HA
  host's local app directory.
- An Android phone with the Volkswagen app already signed in.
- Volkswagen app language set to German or English.
- Either:
  - ADB over Wi-Fi paired from the HA host, or
  - USB debugging with the phone visible to the HA host or VM.

The app does not include shared ADB keys. Authorize the key generated by the HA
host on the phone, or pair Android wireless debugging on the HA host.

## Build The Local App Directory

This is only needed for development or when testing an unpublished local copy.
Normal users should install through the custom add-on repository above.

From the repository root, create a staged Home Assistant app directory:

```powershell
powershell -ExecutionPolicy Bypass -File tools\package_ha_app.ps1 -Clean
```

The staged output is written to:

```text
build/home-assistant/vw-app-connector
```

The staging step copies the current `vw_app_connector.py`,
`mqtt_publisher.py`, `requirements.txt` and app metadata into one directory.
Do not edit the staged copy by hand; edit the repository files and rerun the
packaging script.

## Install On HA OS

Copy the staged directory to the HA host's local app directory. With the SSH
alias `homeassistant`, one byte-safe option from Windows is:

```powershell
cmd /c "tar -C build\home-assistant -cf - vw-app-connector | ssh homeassistant ""sudo rm -rf /addons/vw-app-connector && cd /addons && sudo tar -xf -"""
```

Fix file permissions and reload the Supervisor store:

```bash
ssh homeassistant 'sudo find /addons/vw-app-connector -type f -exec chmod 0644 {} \; && sudo chmod 0755 /addons/vw-app-connector/run.sh /addons/vw-app-connector'
ssh homeassistant 'SUPERVISOR_TOKEN=$(cat /run/s6/container_environment/SUPERVISOR_TOKEN) ha store reload'
```

Install the local app:

```bash
ssh homeassistant 'SUPERVISOR_TOKEN=$(cat /run/s6/container_environment/SUPERVISOR_TOKEN) ha apps install local_vw_app_connector'
```

For updates after changing this repository, rerun the packaging/copy steps and
then rebuild the installed local app:

```bash
ssh homeassistant 'SUPERVISOR_TOKEN=$(cat /run/s6/container_environment/SUPERVISOR_TOKEN) ha apps rebuild local_vw_app_connector'
```

## Configure

Open the app's Configuration tab in Home Assistant and set at least:

- `adb_serial`: USB ADB serial. Required by the connector even when Wi-Fi is
  used as the active transport.
- `adb_mode`: `usb`, `wifi` or `auto`.
- `adb_wifi_address`: Android wireless-debugging connection address as
  `IP:Port` when using Wi-Fi.
- `api_key`: random secret required for `POST /action/*`.
- `vw_spin`: Volkswagen S-PIN, required only for lock/unlock actions.
- `verified_app_version`: Volkswagen app version tested for write actions.

Optional MQTT settings enable read-only MQTT publishing and Home Assistant
discovery. Leave `mqtt_host` empty if Home Assistant will use the REST package
or if you do not want the app to publish MQTT discovery entities. MQTT never
accepts vehicle write commands; write actions remain REST-only through the
authenticated `/action/*` endpoints.

The app defaults to `boot: manual`. Enable automatic start in Home Assistant
only after ADB and `/health` are verified.

## ADB Setup Notes

### Wi-Fi ADB

ADB over Wi-Fi is recommended for HA OS when USB passthrough to the HA VM is
not already configured.

1. Enable Android wireless debugging.
2. Pair from a shell on the HA host or from inside the running app container.
3. Configure `adb_mode=wifi` and `adb_wifi_address=PHONE_IP:CONNECTION_PORT`.
4. Start the app and check `/health`.

The Android wireless-debugging connection port can change after a phone
restart or after disabling wireless debugging. Update `adb_wifi_address` when
that happens.

### USB ADB

USB works only when HA OS can see the phone. In a VM, this may require USB
passthrough at the hypervisor layer. After the phone is visible, authorize the
ADB key shown by the HA host on the phone and configure `adb_mode=usb` or
`auto`.

## Smoke Test

After starting the app, check Supervisor state and the connector endpoints:

```bash
ssh homeassistant 'SUPERVISOR_TOKEN=$(cat /run/s6/container_environment/SUPERVISOR_TOKEN) ha apps info local_vw_app_connector --raw-json'
ssh homeassistant 'curl -sS http://127.0.0.1:9920/health'
ssh homeassistant 'curl -sS http://127.0.0.1:9920/charge'
```

A healthy read-only smoke test has:

- app state `started`,
- `/health` with `adbState: device`,
- expected `adbTransport` (`usb` or `wifi`),
- verified Volkswagen app version,
- no rate-limit cooldown,
- `/charge` returning a fresh JSON response.

`/location` and `/details` perform slower multi-page app reads. Use them only
when you intentionally want to spend background budget for those endpoint
refreshes.

## Home Assistant Entities

The app only packages and runs the connector. It does not replace Home
Assistant entity configuration by itself.

Recommended options:

- Enable MQTT in the app and let Home Assistant MQTT discovery create read-only
  entities automatically.
- Or use the REST package example in
  [`examples/vw_app_connector.yaml`](../../examples/vw_app_connector.yaml) and
  point `CONNECTOR_HOST` at the HA host or `127.0.0.1` when appropriate.

The connector's REST API remains available for evcc, authenticated actions and
other clients.

## Security

- Keep `api_key`, `vw_spin`, MQTT credentials and ADB private keys out of Git.
- Do not expose port `9920` to an untrusted network.
- Location data can include address and coordinates; only publish it to a
  trusted MQTT broker.
- The app stores usage counters, caches, diagnostics and any generated ADB keys
  in Supervisor-managed app storage below `/data`.
